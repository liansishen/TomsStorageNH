# Tom's Simple Storage 1.7.10 Port

[English](README.md)

Tom's Simple Storage 1.7.10 Port 是面向 Minecraft 1.7.10 与 GT New Horizons 环境的 [Tom's Simple Storage](https://github.com/tom5454/Toms-Storage) 轻量移植。

本模组定位为 GTNH 前期的库存管理与简略自动化工具，让玩家可以方便地管理普通容器，但不会替代整合包后期的大型存储和物流系统。本项目是有选择的低版本移植，不计划完整复刻上游的全部功能。

## 必需依赖

- GTNH 分支或兼容版本的 NotEnoughItems
- UniMixins

NEI 集成使用 GTNH NEI API 和客户端 Mixin。兼容版本需要提供 `AutoCraftingManager` 与 `DefaultOverlayHandler`。

NotEnoughCharacters 为可选依赖，用于增强中文与拼音搜索。

## 主要功能

### 存储网络

- 存储驱动器将相邻库存组合成统一的存储网络。
- 存储桥连方块可以延伸库存之间的本地连接路径。
- 存储终端支持集中存取、排序和搜索物品。
- 合成终端提供合成格，并支持 NEI 配方填充与自动合成集成。
- 无线终端可以绑定终端，并在配置的距离及同一维度内远程访问。

### 搜索

- 搜索物品名称和 Tooltip；以 `@` 开头时按所属模组搜索。
- 可选兼容 NotEnoughCharacters 的中文与拼音匹配。
- 单个搜索按钮循环四种预设：
  1. 标准搜索
  2. 自动搜索
  3. 自动搜索并保留文本
  4. 自动搜索、保留文本并同步 NEI
- 搜索状态保存在玩家本地客户端，多名玩家使用同一终端时不会互相覆盖。

### 基础自动化

- 库存线缆只连接其他库存线缆、存储驱动器和基础库存漏斗的网络侧。
- 线缆不会直接连接普通库存、机器、终端或存储桥连方块。
- 一个线缆网络可以连接多个漏斗；如果同一线缆网络出现多个存储驱动器，网络会因冲突而停用。
- 基础库存漏斗每 10 tick 传输一个物品，遵守有面库存限制，并可被红石信号停用。
- 漏斗宽端为输入，窄管口为输出：
  - 将线缆接在宽端时，物品从存储网络输出到窄管口一侧的库存。
  - 将线缆接在窄管口时，物品从宽端一侧的库存输入存储网络。
- 从存储网络输出时必须设置物品过滤器；输入网络时过滤器可选。
- 手持物品右键漏斗可设置过滤器；空手潜行右键可清除过滤器。

### 存储驱动器范围预览

- 空手潜行右键存储驱动器，可以切换彩色球形范围预览。
- 球体表示本地库存与存储桥连方块的扫描半径，库存仍然需要连续的连接路径。
- 线缆网络使用独立的节点数量上限，不受该球形范围限制。

## 快速上手

1. 将存储驱动器贴着需要组合的库存放置，并使用存储桥连方块延伸本地连接路径。
2. 将存储终端或合成终端贴着存储驱动器放置，即可访问完整网络。
3. 如需自动化，从存储驱动器铺设库存线缆，并连接一个或多个基础库存漏斗。
4. 调整漏斗方向，让窄管口朝向物品输出目标。线缆接宽端时从网络输出，接窄管口时向网络输入。
5. 从存储网络输出物品前，先为漏斗设置过滤器。

## 构建

运行仓库自带的 Gradle Wrapper：

```shell
./gradlew build
```

重混淆后的发布 Jar 会生成在 `build/libs` 目录中。
